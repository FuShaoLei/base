title: Android打开浏览器的方式 date: 2022-07-12 tags: [学习笔记]

方法一

一般的做法

Intent intent = new Intent(Intent.ACTION_VIEW, Uri.parese("http://github.com/"));
startActivity(intent);

使用此法时，当只有一个浏览器的时候，那么就会打开这个唯一的浏览器，但有多个浏览器的时候，系统就会调起一个对话框，让用户去选择要使用哪个浏览器，并且是仅此一次，还是总是，如果选择总是的话，那么下一次再用这个方法打开链接的时候，将不会再次询问，而是直接使用这个你选择“总是”的浏览器去打开链接

优点：可以让用户自己去选择并设置默认的浏览器

缺点：如果没有设置默认的浏览器，而且也不想设置，那么每次跳转，都要选择浏览器，有点小烦

方法二

让用户每次都去选择浏览器：

Intent intent = new Intent(Intent.ACTION_VIEW, Uri.parse("https://github.com"));
startActivity(Intent.createChooser(intent,"FuShaoLei"));

使用这个方法，如果你只有一个浏览器或者有默认的浏览器的话，直接打开，否则，每次打开链接，都要用户去选择浏览器，并且没有什么仅此一次打开或者总是打开的选项，这种方法还可以自定义标题

优点：不用每次都去选择仅此一次或者总是的选项，可以自定义标题

缺点：如果没有默认的，那么每次都要选，更烦了

方法三

通过指定包名的做法

比方说我查到了google chrome的包名是com.android.chrome，那么我可以这样做

Intent intent = new Intent(Intent.ACTION_VIEW, Uri.parse("https://github.com"));
intent.setPackage("com.android.chrome");
startActivity(intent);

这样的话，如果有多个浏览器，系统也不会进行询问，而是直接打开链接，但是如果手机内没有指定浏览器的话，就会报错了，比方说我知道opera浏览器的包名是com.opera.browser，但是我的手机里并没有opera浏览器，我直接打开的话，就会：

android.content.ActivityNotFoundException: No Activity found to handle Intent { act=android.intent.action.VIEW dat=https://github.com/... pkg=com.opera.browser }

那么我们可以通过android提供的一个叫做PackageManager的类去查看，小小的封装了一下：

public static boolean isAppInstalled(@Nullable Context context, @Nullable String originPackageName) {
    if (context == null || originPackageName == null || originPackageName.length() == 0)
        return false;
    boolean isInstalled = false;
    try {
        PackageManager packageManager = context.getPackageManager();
        List<PackageInfo> list = packageManager.getInstalledPackages(0);
        if (list != null) {
            for (PackageInfo info : list) {
                if (originPackageName.equals(info.packageName)) {
                    isInstalled = true;
                    break;
                }
            }
        }
    } catch (Exception e) {
        Log.e("=>", "isAppInstalled " + e.getMessage());
    }
    return isInstalled;
}

记得在androidmanifest文件中加入：

    <uses-permission
        android:name="android.permission.QUERY_ALL_PACKAGES"
        tools:ignore="QueryAllPackagesPermission" />

指定的包名不一定是浏览器，有些情况下，可以进入到指定的app内，比如telegram的包名是org.telegram.messenger，那么我在遇到前缀诸如https://t.me/的链接的时候，便可以通过指定包名，直接使用telegram打开，而不是使用浏览器打开

另外，在我的开发过程中，发现如果是打开facebook的话需要一点别的操作，需要在data前加入：fb://facewebmodal/f?href=，才可以打开facebook

而app的包名的获取，可以把app下载下来后，用一个叫做skit的软件去获取，当然也可以通过adb工具去获取，总之都是要把app下载下来之后才能去获取的，当然可能也有可以直接看包名的网站，可是我没找到QAQ，这里就列一下我找到的一些app包名，仅供参考：

    // app
    public static String TELEGRAM = "org.telegram.messenger";
    public static String TWITTER = "com.twitter.android";
    public static String REDDIT = "com.reddit.frontpage";
    public static String DISCORD = "com.discord";
    public static String FACEBOOK = "com.facebook.katana";
    // browser
    public static String FIREFOX = "org.mozilla.firefox";
    public static String UC = "com.UCMobile.intl";
    public static String BRAVE = "com.brave.browser";
    public static String SAMSUNG = "com.sec.android.app.sbrowser";
    public static String DUCKDUCKGO = "com.duckduckgo.mobile.android";
    public static String OPERA = "com.opera.browser";
    public static String CHROME = "com.android.chrome";

使用这个方法的优缺点：

  • 优点：直接打开浏览器，而不是让用户去选择，用户体验一流

  • 缺点：指定软件的包名可能会变
