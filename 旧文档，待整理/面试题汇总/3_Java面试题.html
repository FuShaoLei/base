Java

面向对象的三大特性

  • 封装：隐藏内部实现，保证独立性
  • 继承：使用已有的类来构建新的类
  • 多态：指的是引用变量所指的具体类型要在程序运行时才确定

equals和==的区别

  • == 若是基本数据类型，比较的是值，若是引用数据类型，比较的是内存地址
  • equals比较内存地址

8大基本数据类型

基本类型 位数 字节 默认值
int 32 4 0
short 16 2 0
long 64 8 0L
byte 8 1 0
char 16 2 'u0000'
float 32 4 0f
double 64 8 0d
boolean 1 false

String，StringBuilder和StringBuffer的区别

  • String是字符串常量 不适用于经常要改变值得情况，每次改变相当于生成一个新的对象
  • StringBuilder是线程不安全的
  • StringBuffer是线程安全的

类和对象的区别

  • 类：构造对象的模板或蓝图
  • 对象：类的实例化

重载和重写的区别

  • 重载：多个方法，有相同的名字，不同的参数
  • 重写：子类重写父类的方法

抽象类和接口的区别

  • 抽象类：可以有具体方法实现
  • 接口：不能有具体的方法实现，一个类只能有一个超类，但可以实现多个接口

调用构造器后发生了什么

  1. 如果第一行调用了另一个构造器（使用this(...)，另外，在构造器中调用另一个构造器这个语句也只能写在第一行），则执行另一个构造器
  2. 否则则先后执行：
    1. 静态字段
    2. 静态的初始化块
    3. 所有实例字段初始化为默认值（0，false，null）
    4. 初始化块
  3. 最后才是执行构造器主体代码

4种访问修饰符

  • private：仅对本类可见
  • public：对外部完全可见
  • protected：对本包及所有子类可见
  • 默认（无需修饰符）：对本包可见

线程的状态

  • New：新建
  • Runnable：可运行（可能正在运行也可能没有运行）
  • Blocked：阻塞
  • Waiting：等待
  • Timed waiting：计时等待
  • Terminated：终止

final，finally，finalize的区别

  • final：关键字，当一个类被final修饰，表示不可继承，当一个变量或者方法被final修饰，表示在使用中不可改变
  • finally：try catch的补充，必要的一步
  • finalize：一个方法，finalize()方法是在垃圾收集器删除对象之前对这个对象调用的方法。

保证线程安全的机制

线程安全问题的本质：在某一个线程对资源进行写操作的中途其他线程对这个资源又进行了写操作，导致出现了数据错误

  • synchronized关键字：保证同步性，可以修饰方法，也可以修饰代码块，本质是保证方法或者代码块内部的资源，在同一时间，最多只能有一个线程在访问，以及资源的数据同步
  • volatile关键字：积极的去维护同步，一般用多个线程去操作同一个变量的时候会用到（volatile只对基本类型的赋值操作和对象的引用赋值操作有效）
  • AtomicInteger等关键字：给变量加上原子性和同步性

Java中有哪些引用类型

  • 强引用：就是平常new的一个对象，不会被回收，只有当=null时才可能会被回收
  • 软引用：内存不足了，会回收，主要垃圾回收期没有回收，该对象就可以一直被程序使用
  • 弱引用：生命周期比软引用更短，在垃圾收集器线程扫描到的时候，都会进行回收
  • 虚引用：如果一个对象仅是虚引用，那么他就和没有任何引用一样，在任何时候都可能被回收，虚引用必须和引用队列联合使用。

泛型是一种把明确类型的工作推迟到创建对象或者调用方法的时候才去明确的特殊的类型。
